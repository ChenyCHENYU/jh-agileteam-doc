# 项目目录结构扫描提示词

请按照以下要求扫描并分析当前项目的目录结构：

## 任务目标

扫描指定项目的完整目录结构，并按照以下规范输出格式化的目录树，同时分析各目录和文件的用途。

## 扫描要求

1. **递归扫描**：从项目根目录开始，递归扫描所有子目录
2. **忽略以下目录**：
   - `node_modules/`
   - `.git/`
   - `dist/`
   - `build/`
   - `.vitepress/dist`
   - `.cache/`
   - `.temp/`
   - `coverage/`
   - `*.log` 文件

3. **输出格式**：
   - 使用树状结构展示
   - 使用 `📁` 表示目录
   - 使用 `📄` 表示文件
   - 添加注释说明关键目录/文件的用途

## 输出模板

```
项目名称/
├── 📁 public/                    # 静态资源
├── 📁 src/                       # 源代码
│   ├── 📁 api/                   # API 接口层
│   │   ├── 📁 generated/         # 自动生成目录
│   │   ├── 📄 auth.ts            # 认证相关接口
│   │   └── 📄 index.ts           # 统一导出
│   ├── 📁 components/            # 组件库
│   │   ├── 📁 global/           # 全局组件
│   │   └── 📁 local/            # 局部组件
│   ├── 📁 views/                 # 页面组件
│   ├── 📄 App.vue                # 根组件
│   └── 📄 main.ts               # 入口文件
├── 📄 package.json              # 项目配置
└── 📄 README.md                 # 项目说明
```

## 分析要求

在输出目录结构后，请提供以下分析：

### 1. 目录结构分析
- 列出所有一级目录及其用途
- 识别核心业务目录（如 api、components、views 等）
- 识别配置目录（如 config、types 等）

### 2. 命名规范分析
- 分析组件命名规范（前缀、大小写等）
- 分析文件命名规范
- 分析目录命名规范

### 3. 文件组织模式
- 识别页面组件的组织模式（如三文件模式）
- 识别组件库的组织模式
- 识别类型定义的组织方式

### 4. 与标准结构对比
- 对比当前结构与标准结构的差异
- 列出需要补充或调整的部分

### 目录结构分析

| 目录 | 用途 | 说明 |
|------|------|------|
| `public/` | 静态资源 | 存放不需要处理的静态文件 |
| `src/api/` | API 接口层 | 按模块组织 API 接口 |
| `src/components/global/` | 全局组件 | 使用 C_ 前缀，可在项目中全局使用 |
| `src/components/local/` | 局部组件 | 使用 c_ 前缀，仅在特定页面使用 |
| `src/views/` | 页面组件 | 路由页面组件，采用三文件模式 |

### 命名规范分析

- **组件命名**：全局组件使用 `C_` 前缀，局部组件使用 `c_` 前缀
- **文件命名**：Vue 组件使用 PascalCase，样式文件使用 kebab-case
- **目录命名**：组件目录使用 PascalCase

### 文件组织模式

- **页面组件**：采用三文件模式（index.vue、index.scss、data.ts）
- **全局组件**：包含 index.vue 和 index.scss
- **局部组件**：包含 index.vue 和 index.scss

### 与标准结构对比

| 项目 | 当前结构 | 标准结构 | 差异说明 |
|------|----------|----------|----------|
| types 目录 | ❌ 缺失 | ✅ 需要 | 建议添加 types/modules/ 用于类型定义 |
| hooks 目录 | ❌ 缺失 | ✅ 需要 | 建议添加 hooks/ 用于组合式函数 |

## 注意事项

1. 请确保输出完整的目录结构，不要省略关键目录
2. 对于复杂的目录结构，请使用缩进清晰展示层级关系
3. 添加有意义的注释说明，便于理解目录用途
4. 分析部分要具体、准确，避免模糊的描述
5. 对比分析要指出具体的差异和改进建议
