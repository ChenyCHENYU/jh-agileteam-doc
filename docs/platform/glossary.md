# 术语表

---

## 应用

是项目目标在某一个终端类型上的具体实现，运行在用户端，可以和用户进行交互。目前 JH4J-CODE 平台支持的终端类型包括 **Web、iOS、Android**。

### WEB 应用

基于互联网的应用程序，通过浏览器作为用户界面。建立在 Web 技术（HTML、CSS、JavaScript）上，不需要在用户设备上安装。JH4J-CODE 平台搭建的应用使用 Vue.js 框架实现。

### 移动端应用

在移动设备上运行的应用程序。JH4J-CODE 平台采用跨平台方案，搭建一次便可在 iOS 和 Android 上正常运行。

---

## 页面

JH4J-CODE 平台的页面构成要素：

| 要素 | 说明 |
|------|------|
| 页面编码 | 标识页面唯一性，不可为中文 |
| 页面名称 | 页面的中文名称 |
| 组件描述 | 页面包含的组件信息 |

---

## 模型

一组图形化或可视化的接口，用于展示业务需求和解决方案。

| 类型 | 说明 |
|------|------|
| 单表模型 | 对应数据库一个表，用于单个表就能解决问题的场景 |
| 多表模型 | 对应数据库多个表，适用于多表配合才能完成任务的场景 |

---

## 组件

| 类型 | 说明 | 示例 |
|------|------|------|
| 原子组件 | 最小功能单位，最高复用性 | 按钮、输入框、列表项 |
| 高级组件 | 由原子组件组成的复杂组件 | 表格、表单 |
| 页面组件 | 页面上独立的部分或功能 | 导航栏、页眉、页脚、卡片 |

---

## 业务类型

平台根据业务场景中实际需要的数据类型归类：

| 类型 | 标识 |
|------|------|
| 数据字典 | `dict` |
| 枚举 | `enums` |
| 用户 | `user` |
| 公司 | `company` |
| 部门 | `dept` |
| 日期 | `date`（`yyyy-MM-dd`） |
| 日期时间 | `datetime`（`yyyy-MM-dd HH:mm:ss`） |
| 时间 | `time`（`HH:mm:ss`） |
| 文本 | `text` |
| 长文本 | `textarea` |
| 数字 | `number` |
| 自动编码 | `auto_no` |
| 布尔值 | `boolean` |

---

## 变量

页面中视图的更新通过组件绑定变量实现，变量的改变通过重新赋值。

| 属性 | 说明 |
|------|------|
| 命名（name） | 以英文字母开头，不可为中文，建议有业务含义（如 `userList`） |
| 描述（label） | 中文名称 |
| 数据类型 | string / number / boolean / array / object / function / date |
| 默认值 | 变量初始值 |
| 业务类型 | 对应的业务数据类型 |
| 业务值 | 业务类型下的枚举值 |

---

## 角色

针对数据权限的分组方式，控制拥有该角色的用户可查看的业务数据。

---

## 页面路由

页面跳转路由，为需要跳转的页面配置路由规则，当满足路由规则时由当前页面跳转到目标页面。
