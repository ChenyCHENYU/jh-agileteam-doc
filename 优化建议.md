# 文档站优化建议

## 1. 深色模式适配

AuthorTag 和 TeamHero 组件里存在硬编码的 `rgba(255,255,255,...)` 值，深色模式下会穿帮，建议全部替换为 VitePress CSS 变量（`var(--vp-c-*)`）。

涉及文件：
- `docs/.vitepress/components/AuthorTag/index.scss`
- `docs/.vitepress/components/TeamHero/index.scss`

---

## 2. 侧边栏规划中状态标注

L5 / L6 / L7 页面是"规划中"状态，但在侧边栏与已实现页面外观完全一致，用户点进去才知道是空页面，体验差。

建议在 `.vitepress/config.ts` 的 sidebar 配置中，给这些项添加 `badge` 标注：

```ts
{ text: 'L5 — Agent Pipeline 🚧', link: '/views/best-practices/L5-agent-pipeline' }
```

或使用 VitePress 内置 `badge` 配置项（如果版本支持）。

---

## 3. BackendTeamHero 风格同步

前端团队页（TeamHero）刚完成重构：移除职级标签、新增 `spotlight` 贡献描述、bio 引号斜体、渐变头像环。

`BackendTeamHero` 目前是旧风格，两个团队页放在一起会割裂。

涉及文件：
- `docs/.vitepress/components/BackendTeamHero/index.vue`
- `docs/.vitepress/components/BackendTeamHero/data.ts`
- `docs/.vitepress/components/BackendTeamHero/index.scss`（如有）

---

## 4. 中文搜索体验

VitePress 默认使用 minisearch，中文分词效果差，搜索"工程化"可能搜不到"AI工程化"相关内容。

建议接入 **Algolia DocSearch**（开源项目免费配额），申请地址：https://docsearch.algolia.com/apply/

长期方案也可考虑 VitePress 的 `search.provider: 'local'` 配合 `_tokenize` 自定义分词器。

---

## 5. 首页数字动效（低优先级）

GlassHome 首页可以考虑加入几个核心数字动画卡片（如「13人团队 / 9个Skill / 10个MCP Tool」），配合打字机动效标题，3秒内让访客明白这个文档站是干什么的。
