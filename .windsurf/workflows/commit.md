description: Git 提交规范，所有代码提交必须遵循此格式
---

## 提交格式

```
<type>(<scope>): <简短描述>
```

## type 可选值

| type | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 Bug |
| docs | 文档变更 |
| style | 样式调整（不影响逻辑） |
| refactor | 重构（非新功能、非修复） |
| perf | 性能优化 |
| chore | 构建/工具链变更 |

## scope 示例

- 前端skill、后端skill、团队页、首页、导航、侧边栏、样式、组件 等

## 规则

1. 描述精简，一句话说清改了什么，不超过 50 字
2. 中文描述优先
3. 不加句号结尾
4. 一次提交只做一件事，不混合多个不相关改动（除非是同一批次的初始建设）

## 示例

```
docs(前端skill): 新增 5 个 AI Skill 文档及流水线总览
feat(团队页): 新增后端团队成员展示组件
style(导航): 优化亮色模式下不可选菜单项样式
fix(表格): 修复列宽溢出导致文字换行
```
